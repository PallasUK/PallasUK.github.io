What is DevSecOps?

The DevOps movement was not started from a single manifesto, instead it grew from a desire to improve on the status quo. For many years development and operations teams operated as seperate isolatated teams. This meant developers would not understand the needs of a production environment and operations staff would not trust software produced by developers.

The goal was to break down the barriers between teams operations staff would provide development environments which reflect the production environments with the same constraints which would allow the same tooling to be used in both environments. This allows developers to better understand the target environment reducing integration and deployment issues.

However it isn't enough to deploy an application quickly, you need to ensure the environment is secure and the application isn't exposing any vulnerabilities. Traditional development would place reviews with quality assurance and information security before moving to production. With the faster pace of change these teams need to be incorporated into the process and involved from design up to deployment. This has lead to the term DevSecOps.
